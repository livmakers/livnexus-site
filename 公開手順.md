# リブネクサス株式会社サイト 公開手順（ドメイン取得 → 正式公開）

## 🟢 現状（2026-06-05 時点）

- **仮公開 稼働中**：https://livmakers.github.io/livnexus-site/ （GitHub Pages・昼テーマ完成版）
- **独自ドメイン接続：GitHub側 完了**（`livnexus.co.jp` を設定済み）
- **残りはこれだけ → ドメインのDNS設定**（下記「▶ いますぐやること」）

## ▶ いますぐやること：DNS設定（レジストラの管理画面）

`livnexus.co.jp` を買ったサービス（お名前.com 等）の **DNSレコード設定**で、以下を登録してください。

| 種類 | ホスト名 | 値（向き先） |
|---|---|---|
| A | @（または空欄＝ルート） | `185.199.108.153` |
| A | @ | `185.199.109.153` |
| A | @ | `185.199.110.153` |
| A | @ | `185.199.111.153` |
| CNAME | `www` | `livmakers.github.io` |

- 4本の A レコードはすべて登録します（GitHub Pages の冗長IP）。
- 反映は数分〜最大48時間。反映後、`https://livnexus.co.jp` で表示されます。
- SSL（https）は GitHub が自動発行 → その後「Enforce HTTPS」を有効化（私が確認します）。
- 確認：`https://dnschecker.org` に `livnexus.co.jp` を入れて A が 185.199.x.153 になればOK。

## 全体像（5ステップ）

```
[コード] GitHub  →  [自動ビルド & 公開] Vercel  →  [独自ドメイン接続] レジストラのDNS
```

- **費用感（年間）**：ドメイン .co.jp 約3,000〜4,000円／年 ＋ Vercel 無料（Hobbyプラン）
- **所要時間**：30分〜1時間（ドメインのDNS反映待ちを除く）

## STEP 1. GitHub にコードを上げる ✅ 完了

- リポジトリ：**https://github.com/livmakers/livnexus-site** （public）
- 以後の更新は `git add -A; git commit -m "..."; git push` で反映 → Vercel が自動再デプロイ。

---

## STEP 2. Vercel にデプロイ（無料）

1. [https://vercel.com](https://vercel.com) を開き、**「Continue with GitHub」**でサインアップ（無料 Hobby プラン）。
2. **Add New… → Project** をクリック。
3. リポジトリ一覧から **`livnexus-site`** を選び **Import**。
4. Framework Preset が **Next.js** に自動判定されるのを確認し、そのまま **Deploy**。
5. 1〜3分で `https://livnexus-site-xxxx.vercel.app` の仮URLで公開されます（まず動作確認）。

> 以後、GitHub に変更を push するたびに Vercel が自動で再ビルド・再公開します。

---

## STEP 3. ドメインを取得 ← 採用：`livnexus.co.jp`

- **採用ドメイン：`livnexus.co.jp`**
  - グループ各社（artnap.co.jp / tnk-tanaka.co.jp / web-yamada.co.jp）と表記を統一でき、ブランドの一貫性◎
  - `.co.jp` は日本で登記された会社のみ取得可能＝**信頼感が高い**
- レジストラ（どこで買うか）の例：
  - お名前.com / ムームードメイン / Xserverドメイン / Value Domain など
- 代替案：`livnexus.com`（国際展開を見据える場合）／`livnexus.jp`

---

## STEP 4. ドメインを Vercel に接続

1. Vercel の対象プロジェクト → **Settings → Domains** → 取得したドメイン（例 `livnexus.co.jp`）を **Add**。
2. Vercel が表示する DNS レコードを、**レジストラ側のDNS設定**に登録：
   - ルート（`livnexus.co.jp`）→ **A レコード** `76.76.21.21`
   - `www` → **CNAME レコード** `cname.vercel-dns.com`
   - ※ 実際の値は Vercel の画面に表示されるものを使用してください。
3. DNS 反映後（数分〜最大48時間）、**SSL（https）は Vercel が自動発行**。`https://livnexus.co.jp` で公開完了。

## （任意）本番をさらに強化したい場合

四位一体監修で挙がった「将来の強化候補」：

1. **独自ドメインメール**：`info@livnexus.co.jp` を作成し、問い合わせ先を個人Gmailから差し替え（`src/config/company.ts` の `contactEmail`）。
2. **フォームのサーバー送信化**：現状の mailto 方式 → [Resend](https://resend.com) / [Formspree](https://formspree.io) 等のAPIに接続（送信記録が残り、取りこぼし防止）。
3. **計測**：Google Search Console / Google Analytics を登録（SEO・流入分析）。
4. **リブリンク株式会社のリンク**：URL確定後 `GROUP_PARTNERS` に追記。

---

## 困ったとき

- Vercel のビルドが失敗する → ビルドログを確認（多くは `npm install` の依存解決。`package.json` は確認済み）。
- ドメインが繋がらない → DNS 反映待ち（最大48時間）。`https://dnschecker.org` で反映状況を確認。
